Stanford CS242 assign8 - Typestate shopping cart (Rust, session/typestate types)
Built + run in WSL Ubuntu, cargo/rustc 1.96.1.

The `Cart<S>` struct is parameterized by a phantom state marker (Empty /
NonEmpty / Checkout). Each state exposes only the operations legal in it, so
illegal transitions (e.g. checking out an empty cart) are rejected by the type
checker at COMPILE time, not at run time. The `cart_fail` test uses `trybuild`
to assert that a program which calls `.checkout()` on a `Cart<Empty>` fails to
compile with exactly:

    error[E0599]: no method named `checkout` found for struct
                  `Cart<Empty>` in the current scope
      = note: the method was found for `Cart<NonEmpty>`
